①githubへアップするフォルダにGIT設定をする
　　GitHubのページで『New Repository』を作成します。
　　名前は例えば『my_extension』などとします。
　　初期ファイル（READMEなど）は作らずに、空のリポジトリを作成します。

②ローカルの拡張機能フォルダをgitで初期化
　　拡張機能フォルダ（例：my_extension）に移動して、次の操作を行います：
　　　　cd my_extension
　　　　git init
　　　　git add .
　　　　git commit -m "initial commit"
　　　　git branch -M main
　　　　git remote add origin https://github.com/yourname/my_extension.git
　　　　git push -u origin main
　　これで最初のアップロードが完了します。

③コード修正時のアップロード方法
　　ローカルでファイルを変更した後は、以下の手順ですぐにアップロードできます。

　　（１）ローカルの変更をコミットする
　　　　　修正したフォルダ内で：
　　　　　　　git add .
　　　　　　　git commit -m "修正内容を簡潔に記述"　
　　　　　※コミットメッセージは「何を修正したか」が後からわかるようにしましょう。
　　（２）GitHubにpushする
　　　　　　　git push origin main
　　これだけで完了します。

〇MakeCode側でのキャッシュ問題への対策
　　GitHub URL末尾にクエリパラメータを追加（簡易方法）
　　　　https://github.com/yourname/my_extension?update=1234

　　MakeCodeのキャッシュを手動で消す（補助方法）
　　　　ブラウザで「Ctrl + Shift + R」（Windowsの場合）で強制リロード
